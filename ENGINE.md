## 定位
**局部规则 → 整体行为**这一条因果链本身。机制优先、**动机先行**（从读者能 connect 的现象切入——堵车、谣言、停电、崩盘——不从方程或术语开场）。分工：mathematics=数学语言（分形/动力系统/图论/信息论），physics=物理实现（相变/临界/重整化），mental-models=单个模型的应用话术，system-design=工程实践，investing=金融应用——本站讲**机制**：简单规则怎么长出整体行为、这种行为为什么预测不了、以及能拿它怎么办。

## 两类页面

- **参考库** `ref-{slug}.html`（+ `.en.html`）：**经典模型/方法**的独立解释页，通用、不绑定某一期（沙堆模型、谢林模型、SIR、Ising……）。**正文碰到某个模型就用 pill 链接 `<a class="reflink" href="ref-xxx.html">名称</a>` 链过去；若该 ref 页还不存在，就顺手把它也生成出来**（放进 index 参考库栏）。参考页也要双语。

## 文风（面向零基础，硬规则）
- **所有术语都当读者完全不懂，随写随用大白话解释**——不假设任何数学、物理、编程背景（别写"你应该学过""众所周知"）。
- 口语化、松弛；从日常场景切入（堵车怎么凭空出现、一句谣言怎么破圈、为什么排队总是另一队快）。
- 短句、多"你"（仅用于人人都有的共同经验）。
- 无元叙述：不写"我给参考页链接""这一期我们讲…"这类话，正文只留内容。
- **不许炫术语**。复杂性科学最大的病是隐喻当解释——每提一个概念，必须能说清它**具体预测什么、什么情况下不成立**。说不清就别写进去。
- **中文页的术语要挂英文原名**（英文页不需要）：
  - **小标题**：`<h2>` 里跟一个 `<span class="he">（English Title）</span>`，小字暗色，不抢正文。
  - **重要概念首次出现时**在括号里给英文原名——幂律（power law）、吸引子（attractor）、无标度（scale-free）、临界慢化（critical slowing down）。只在**首次**给，后面别反复挂。这是为了读者能拿它去搜英文资料，不是为了显得专业。
  - **缩写必须先定义再使用**，一次都不许提前。写 SOC 之前必须出现过「自组织临界性（self-organized criticality），缩写 SOC」。这是最容易犯的错——你自己写惯了就忘了读者没见过。

## 图（硬规则）
- **一律画 SVG，绝不用等宽 ASCII 图。** 视觉风格贴合全站暗色调、与基准页协调即可。
- **SVG 图必须清晰、易读、准确表达内容**：布局不挤不乱、文字不重叠不出界、箭头/连线渲染后都真实可见、颜色对比够。
- 本站尤其吃图：**能画出「同一规则跑几步后的三帧对比」就画**（生命游戏三代、沙堆的一次雪崩、网络加边到渗透阈值的前后、幂律 vs 正态的同轴对比）。抽象概念一律落到一张能看的图上。
- **箭头一律用 `marker-end` + `orient="auto"`，别手画三角当箭头。** marker 用 `markerUnits="userSpaceOnUse"` 定绝对尺寸。
- **连杆（线/路径）绝不套 `filter` 辉光或 `objectBoundingBox` 渐变。** 轴对齐直线包围盒宽或高为 0，filter 区域和渐变会退化成零 → **整条线不渲染，只剩箭头浮着**。要辉光就在底下单独垫一条无 marker 的粗线做衬，连杆本身用纯色。
- **画完必须真渲染检查**（headless Chromium 截图看一眼）：箭头都贴线、朝向对、无浮空三角、无消失的连杆。

## 每期结构
1. header：`week-tag`（TOPIC N · PHASE X）+ 渐变 h1 + 副标题 + 日期 + pitch（一句反直觉钩子）
2. intro（`.intro`）：why this matters，一个反直觉转向，无元叙述
3. **3–4 个概念 section**，每节：
   - 正文（松弛、零基础）
   - 需要时插 SVG figure
   - 碰到模型/方法 → `→ 参考` pill 链接
   - **`.en-brief`**：一句英文简介（`<p class="en-brief"><b>EN</b> …</p>`）总结该节核心。**只在中文页有；英文页去掉这些块**。
   - **`.decide` 🎯 决策线**（铜色框 `#e0895e`）：这条机制对**预测、干预、下注**意味着什么——一个具体可执行的动作或一个该停掉的动作。**每期至少一个，可以多个**；宁可少而实，别写成鸡汤。
   - **`.crossx-inline` 🌀 节内越界**（细绛色左边框 + 小字，**不是色块**，紧跟在本节正文之后）：**默认 1 条，2–3 句。**
     - 必须挂**本节这一条机制**——不许借隔壁节的机制，不许提前用文末大框的料。
     - 联想要在读者刚看完机制时立刻投递，这是它比放文末强的全部理由；所以写法要短、要即插即用。
     - **允许缺席**：纯定义/纯铺垫的节挂不上就别硬凑。某节连一条都导不出去，通常说明这节的机制本身写含混了——先回去改机制，别改这条规则。
     - **可以写 2 条，但要过准入**：两条必须**分属不同大类**，且各自推出**不同的结论**。同一个结论举两个例子 = 不是两条，是一条没选好——挑更硬的那个，删掉另一个。做不到就老实写 1 条。
     - **不许 3 条及以上**。到 3 条它已经是个板块而不是余韵了，节奏就毁了。真有 3 条好料，说明要么这节其实该拆成两节，要么该走下面的路由规则送去文末。
     - **路由规则（比数数更重要）**：候选超过 1 条时，先问「**它只用到本节这一条机制，还是同时用到了本期别的机制？**」——只用本节的 → 留节内；用到两条以上的 → **送去文末大框**，那正是大框要的成套论证。所以「该写几条」多半不是取舍题，是分拣题。
     - **页级预算**：全页节内越界总数 **≤ 节数 + 1**。也就是至多允许一节写 2 条，其余保持 1 条。超了就往文末挪，别扩军。
4. **`.bigcat` 🎒 场景 · BigCat**（沙金框 `#d9b45f`，放在最后一节之后、越界大框之前）：**每期一个，2–3 条**，编号 ①②③。

   **与决策线的分工（分不清就是白写）**：`.decide` 是**这条机制本身蕴含的行动含义**，对谁都成立；本框是**落到 BigCat 具体处境里的那几个位置**——在哪件反复发生的事情上、该改看什么、该停掉什么。整期只有这一个框谈"你"，所以它必须具体到能对号入座。

   **硬要求**：
   - 每条**先点领域**，再给场景。领域池：`工程与系统设计` · `带团队与组织` · `投资与仓位` · `育儿` · `修行与心性` · `写作与这个学习站本身` · `健康与精力`。**同一领域别连续三期出现**，写前 `grep -h 'bigcat' *-topic*.html | tail -30` 扫一眼。
   - **必须是一个反复发生的具体情境**，不是一个类别。「带团队」✗ ／「季度目标定完之后，没人愿意第一个说进度已经偏了」✓。
   - **必须落到可改的东西上**：改看哪个指标、改问哪句话、停掉哪个动作。给不出就换一条。
   - **禁止把 `.decide` 换个说法重讲一遍**。如果某条抽掉"BigCat"三个字后和决策线没区别，删。
   - 不许假设 BigCat 的年龄、性别、家庭构成、职级等未知细节；只用仓库体系里明确成立的面向（AI/工程、组织、投资、育儿、佛学、写作、健康）。
   - 宁可 2 条硬的，不要 3 条凑数。
5. **`.crossx` 🌀 越界 · 跨学科的联想**（绛色框，放"深入思考"之前）——**本站的另一条主线，不是可选装饰。**

   复杂性科学的全部分量就在于：**同一条机制反复出现在互不相干的领域**。

   **⚠️ 关于 TOPICS.md 里的 `🌀 锚`**：那是**一条方向**，不是待扩写的条目，也不是答案。它存在的唯一理由是注入 BigCat 的口味、防止每期都漂到佛道去。**联想必须由你在写的时候现场找**——因为那时你手上有本期实际写出的机制、实际选的例子、实际的段落语境，这才是联想真正要接住的东西；预先想好的清单接不住。**锚可以只用一半，也可以完全不用**（只要你找到了更好的）。**把锚撑成句子交差 = 这件事你没做。** 每期至少要有一条是锚里完全没提到的。

   **本框的工作与节内越界不同，绝不是把节内的再说一遍**：节内是「**这一条**机制也在那儿」；本框是「本期**这几条机制作为一整套**在那个领域同时成立」，以及**边界在哪**。后者是节内视角做不到的，也才是复杂性科学真正的主张。

   **硬要求**：
   - **每期 2–3 条**（`<ul><li><b>学科·人物</b>：…</li></ul>`），每条 3–5 句。少而深，别扩军。
   - **必须领域跨得开**：**整页统计**（节内 + 本框一起算）至少覆盖三大类，不是每节各自跨。七大类——① 东方思想（佛/道/儒/易）② 西方哲学与科学哲学 ③ 历史与军事 ④ 经济与制度 ⑤ 文学与艺术 ⑥ 生物/医学/神经 ⑦ 工程与技术史。
   - **必须点明共通的机制，而不是共通的感觉**。写法固定：*那个领域的现象 → 它背后跑的是本期哪条机制 → 因此能推出什么原本看不出的结论*。做不到第三步的联想不许写。
   - **必须标注失配处**：至少一条要写「**但这里不成立**：…」——那个领域在哪一点上不满足本期机制的前提。这是本框与"文青式类比"的唯一分界线。**边界服务于共通点**：写失配是为了让站得住的那部分更硬，不是为了显得谨慎，所以别把整条写成免责声明。
   - **不许重复吃老本**：佛学/道家很容易每期都挂，写之前先 `grep -h 'crossx' *-topic*.html | tail -40` 扫一眼前几期挂了谁，同一人物/典故连续三期内不重复出现（节内与文末合并计算）。
   - **宁可硬，不可软**。「都是一种循环」「都体现了整体大于部分」这种写法直接删掉重写。

   **反例**（不许这样写）：「佛教讲无常，混沌也讲变化，都说明世界不确定。」——没有机制、没有新结论、没有边界。

   **节内越界正例**（短、即时、单机制）：「Milgram 的六度实验其实只有约 5% 的信件送达。'六度'算的是**成功路径**的长度，失败的那 95% 不进分母——所以它描述的是网络里存在捷径，不是人人都够得着。」

   **文末大框正例**（长、成套、带边界）：「斯多葛把世界切成'可控/不可控'两半。本期三条机制合起来说明这条线画错了地方：混沌意味着**李雅普诺夫时间之内**你的干预真能改变轨迹、之外只是在放大误差；分岔意味着**逼近临界点时**同样的力气效果剧增；临界慢化意味着**这个窗口可以被监测**。所以'可控'不是按事情的种类分的，是按**你离临界点多远、还剩多少时间**分的——这是斯多葛给不出的操作性。**但这里不成立**：斯多葛的二分是伦理主张，本来就不承诺可预测性，它要解决的是执念而不是预测；用动力学去'纠正'它是越界的。能借走的是那把尺子，不是它的结论。」
6. `.thinking` 深入思考：3–5 个 `<details>` 开放问题。
7. `.refs` 延伸阅读：**每条都要可点链接**（维基词条 / arXiv / 出版社，别放纯文本）。

## 自检
